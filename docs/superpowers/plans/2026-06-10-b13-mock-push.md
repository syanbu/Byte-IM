### Step 5：Android — WorkManager 接入

- 修改 `app/build.gradle`：
  - `implementation "androidx.work:work-runtime-ktx:2.9.1"`。
- 新增 `app/src/main/java/com/buyansong/im/push/PushPollWorker.kt`：
  - `class PushPollWorker(ctx, params) : CoroutineWorker(ctx, params)`。
  - `doWork()` 流程：
    1. `authRepository.ensureValidSession()`，null → `Result.success()`。
    2. 读 `lastSeenPushId` from `MockPushTokenStore`。
    3. GET `/push/pending?since=&limit=50`。
    4. 解析 `pending[]`；空 → `Result.success()`。
    5. 对每条构造 `NotificationCompat.Builder(this, CHANNEL_ID)`，post。
    6. POST `/push/ack`。
    7. 持久化 `lastSeenPushId`。
    8. `Result.success()`；网络异常 → `Result.retry()`。
  - `inputData`：可空，预留 userId override（多账号切换用）。
- 新增 `app/src/main/java/com/buyansong/im/push/PushPollScheduler.kt`：
  - `object PushPollScheduler { const val WORK_NAME_PREFIX = "push-poll-" }`。
  - `fun schedule(context, userId)`：`enqueueUniquePeriodicWork(WORK_NAME_PREFIX + userId, KEEP, PeriodicWorkRequestBuilder<PushPollWorker>(15, MINUTES).setConstraints(Constraints.Builder().setRequiredNetworkType(NetworkType.CONNECTED).build()).build())`。
  - `fun cancel(context, userId)`。
- 触发点：
  - `ImApp.onCreate`：调度一次（worker name 用 `last-known-user-id`，从 SharedPreferences 读；首次安装 userId 未知时不调）。
  - `AuthenticatedImNavHost` `DisposableEffect`：start 时 `schedule(this@context, session.userId)`。
  - `LoginViewModel.logout()`：取消。
- 通知 channel 创建：
  - `ImApp.onCreate` 中 `createNotificationChannel(NotificationChannel("im_messages_push", "新消息提醒", IMPORTANCE_HIGH).apply { description = "新消息提醒" })`。

### Step 8：手工 E2E 验证

1. 启动 mock-server（`./start-mock-server.sh`）。
2. 安装 Android 客户端，登录 user A 和 user B 各一端。
3. 验证两端 WebSocket 均为 `Authenticated`。
4. 验证 WorkManager 周期任务已调度：

   ```bash
   adb shell dumpsys jobscheduler | grep com.buyansong.im
   ```
5. user A → user B 发文本，user B 在线 → ChatScreen 立刻显示（WebSocket 主路径，**不弹** push 通知）。
6. user B 端按 Home 退到后台，然后使用 `adb shell am kill com.buyansong.im` 模拟后台进程被系统回收；不要使用 `force-stop`，因为 Android 会阻止该应用的 WorkManager/JobScheduler 继续运行。
7. user A 再发文本给 user B。
8. 等 15 min（WorkManager 周期）；或手动触发：

   ```bash
   adb shell am broadcast -p com.buyansong.im -a com.buyansong.im.DEBUG_RUN_PUSH_POLL
   ```
9. 验证 user B 设备系统通知栏出现 "user A: <text>"。
10. tap 通知 → App 冷启动 → ChatScreen 打开，消息已入库。
11. 杀 user B → user A 撤回该消息 → 触发 poll → 通知内容 = "[撤回了一条消息]"（首版可只显示"您有一条新消息"）。
12. 群聊：杀 user B → user A 在群发 → 触发 poll → user B 收到通知。
13. 多设备同 userId：user B 在 A 端再登录，token 覆盖；user A 端 push 不再收到（B13 多端简化方案）。
