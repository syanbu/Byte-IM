# 项目背景与已完成功能 Prompt

你现在接手的是一个本地自研 IM 项目，目标是实现一个运行在 Android 上的即时通讯客户端，并配套一个本地 Java/Netty mock server 作为联调和验证环境。Android 端使用 Kotlin 开发，当前是单模块工程；消息、本地会话、待重试消息、用户资料等数据持久化使用 SQLite 直接实现，没有引入 Room。服务端主要承担本地账号注册登录、HTTP 鉴权、WebSocket 长连接、消息转发、ACK、离线暂存和本地测试支持。这个项目当前聚焦于单聊、本地联调、自定义协议、连接可靠性和基础资料能力，不包含群聊、图片消息、撤回、已读回执、推送等后续扩展范围。

项目当前已经完成的能力如下：Android 端已经完成手机号注册登录、双 token 鉴权、登录态恢复、refresh token 刷新、登出清理；已经完成基于 WebSocket 的单聊文本消息发送与接收；已经完成会话列表、未读数、最后一条消息预览、进入会话清未读；已经完成 SQLite 本地消息持久化、本地历史分页加载、会话数据存储、待发送消息 outbox 持久化；已经完成自定义二进制协议编解码；已经完成连接生命周期管理，包括前台 15 秒心跳、后台 75 秒心跳、心跳超时检测、断线重连、指数退避和重连状态展示；已经完成消息可靠性第一阶段，包括发送后等待 `MESSAGE_ACK`、超时重试、失败标记、重复消息幂等处理；已经完成接收侧 `DELIVERY_ACK`；已经完成消息顺序处理，区分 `clientSeq` 和服务端 `serverSeq`；mock server 已支持本地注册登录、WebSocket 鉴权、消息转发、ACK、离线消息暂存、消息持久化和重启后恢复；客户端 UI 侧已经完成 Messages/Me 等主入口、聊天页、联系人入口、资料展示与编辑、头像上传链路、会话和聊天中的昵称头像展示，以及返回导航语义修正。
